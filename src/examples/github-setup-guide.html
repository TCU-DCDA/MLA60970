What is GitHub?

GitHub is a web-based platform that uses Git for version control. It allows you to store your code online, collaborate with others, track changes, and showcase your projects. Think of it as Google Drive for developers!

GitHub Workflow Overview

Local Code
Write on your computer
Stage Changes
git add
Commit
git commit
Push to GitHub
git push

Step 1: Create a GitHub Account

Account Setup

  1. Go to github.com
  2. Click "Sign up" in the top right
  3. Choose a professional username (e.g., "john-doe" or "jdoe-dev")
  4. Use your school email address
  5. Create a strong password
  6. Verify your account via email
üí° Username Tips:
  • Use lowercase letters and hyphens
  • Avoid numbers unless meaningful
  • Keep it professional (employers will see this!)
  • Make it memorable and easy to type

Student Benefits

As a student, you get GitHub Pro for free!

  1. Go to GitHub Student Developer Pack
  2. Click "Get your pack"
  3. Verify your student status with your .edu email
  4. Upload a photo of your student ID if needed

Step 2: Install Git

macOS Installation

# Option 1: Install via Homebrew (recommended) brew install git # Option 2: Install via Xcode Command Line Tools xcode-select --install # Verify installation git --version git version 2.39.0

Windows Installation

  1. Download Git from git-scm.com
  2. Run the installer with default settings
  3. Choose "Git Bash" as your terminal
  4. Open Command Prompt or Git Bash and verify:
git --version git version 2.39.0.windows.1

Step 3: Configure Git

Before using Git, you need to tell it who you are. This information will be attached to your commits.

# Set your name (use your real name) git config --global user.name "Your Full Name" # Set your email (use the same email as your GitHub account) git config --global user.email "your.email@example.com" # Set default branch name to 'main' git config --global init.defaultBranch main # Verify your configuration git config --list user.name=Your Full Name user.email=your.email@example.com init.defaultbranch=main

Step 4: SSH Key Setup (Recommended)

SSH keys provide a secure way to connect to GitHub without entering your password every time.

Generate SSH Key

# Generate a new SSH key ssh-keygen -t ed25519 -C "your.email@example.com" # When prompted for file location, press Enter for default # When prompted for passphrase, press Enter (or add one for extra security) # Start ssh-agent eval "$(ssh-agent -s)" # Add your SSH key to ssh-agent ssh-add ~/.ssh/id_ed25519

Add Key to GitHub

  1. Copy your public key:
# macOS pbcopy < ~/.ssh/id_ed25519.pub # Windows (Git Bash) clip < ~/.ssh/id_ed25519.pub # Linux cat ~/.ssh/id_ed25519.pub
  1. Go to GitHub ‚Üí Settings ‚Üí SSH and GPG keys
  2. Click "New SSH key"
  3. Give it a title (e.g., "My Laptop")
  4. Paste your key and click "Add SSH key"

Writing Good Commit Messages

Clear commit messages help you and others understand what changed and why.

‚ùå Bad Commit Messages

git commit -m "fix" git commit -m "update" git commit -m "stuff" git commit -m "asdf" git commit -m "final version" git commit -m "final version 2"

‚úÖ Good Commit Messages

git commit -m "Add navigation menu to header" git commit -m "Fix responsive layout on mobile devices" git commit -m "Update contact form validation" git commit -m "Add CSS animations to hero section" git commit -m "Remove unused JavaScript functions"
üí° Tips for good commits:
  • Start with a verb (Add, Fix, Update, Remove)
  • Keep it under 50 characters
  • Be specific about what changed
  • Use present tense

Essential Git Commands Cheat Sheet

# Repository basics git clone [url] # Download a repository git status # Check what's changed git log --oneline # See commit history # Making changes git add [file] # Stage specific file git add . # Stage all changes git commit -m "message" # Commit staged changes git push # Upload to GitHub # Getting updates git pull # Download latest changes git fetch # Check for updates (doesn't merge) # Branching (advanced) git branch # List branches git checkout -b [name] # Create new branch git merge [branch] # Merge branch

GitHub Pages: Host Your Website for Free

GitHub Pages lets you turn any repository into a live website!

Enable GitHub Pages

  1. Go to your repository on GitHub.com
  2. Click the "Settings" tab
  3. Scroll down to "Pages" in the left sidebar
  4. Under "Source", select "Deploy from a branch"
  5. Choose "main" branch and "/ (root)" folder
  6. Click "Save"
  7. Your site will be available at:
    https://username.github.io/repository-name

Troubleshooting Common Issues

üö® Common Problems

# Problem: "Permission denied (publickey)" ssh -T git@github.com # Solution: Check SSH key setup # Problem: "Author identity unknown" git config --global user.name "Your Name" git config --global user.email "email@example.com" # Problem: "Repository not found" git remote -v # Check if URL is correct # Problem: Merge conflicts git status # Edit conflicted files, then: git add . git commit -m "Resolve merge conflict"
