Week 1 Semantic HTML Guide

What is Semantic HTML?

Semantic HTML means using HTML elements for their intended purpose, not just their appearance. This makes your content more accessible to screen readers, search engines, and other technologies that process web content.

Document Structure Elements

These elements define the overall structure of your webpage:

<header> <nav>...</nav> </header> <main> <section> <article>...</article> <aside>...</aside> </section> </main> <footer>...</footer>
  • <header> - Top section with site title, navigation, or introduction
  • <nav> - Navigation links
  • <main> - Primary content of the page
  • <section> - Thematic groupings of content
  • <article> - Self-contained, independently distributable content
  • <aside> - Sidebar content or supplementary information
  • <footer> - Bottom section with copyright, links, contact info

Content Organization Elements

✅ Semantic (Good)

<article>
  <header>
    <h1>Article Title</h1>
    <time datetime="2025-01-07">January 7, 2025</time>
  </header>
  
  <p>Article content goes here...</p>
  
  <footer>
    <p>By: <address>Author Name</address></p>
  </footer>
</article>

❌ Non-semantic (Avoid)

<div class="article">
  <div class="header">
    <div class="title">Article Title</div>
    <div class="date">January 7, 2025</div>
  </div>
  
  <div class="content">Article content...</div>
  
  <div class="footer">
    <div>By: Author Name</div>
  </div>
</div>

Heading Hierarchy

Use headings in proper order to create a logical document outline:

<h1>Page Title (only one per page)</h1> <h2>Major Section</h2> <h3>Subsection</h3> <h4>Sub-subsection</h4> <h2>Another Major Section</h2>

Don't skip heading levels! Go from h1 to h2 to h3, not h1 to h3.

Text Meaning Elements

Emphasis & Importance

  • <strong> - Important text (bold)
  • <em> - Emphasized text (italic)
  • <mark> - Highlighted text
  • <small> - Fine print

Technical Content

  • <code> - Inline code
  • <pre> - Preformatted text
  • <kbd> - Keyboard input
  • <var> - Variables

References & Citations

  • <cite> - Citation of work
  • <q> - Short inline quotes
  • <blockquote> - Long quotes
  • <abbr> - Abbreviations

Form Elements

Properly label form elements for accessibility:

<form>
  <fieldset>
    <legend>Contact Information</legend>
    
    <label for="name">Full Name:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="message">Message:</label>
    <textarea id="message" name="message"></textarea>
    
    <button type="submit">Send Message</button>
  </fieldset>
</form>

Benefits of Semantic HTML

🔍 SEO Benefits

  • Search engines understand your content better
  • Improved search result rankings
  • Rich snippets and featured results

♿ Accessibility

  • Screen readers can navigate efficiently
  • Better keyboard navigation
  • Clear content structure for all users

🛠️ Developer Experience

  • Easier to understand and maintain
  • Better CSS targeting options
  • Consistent across team members
